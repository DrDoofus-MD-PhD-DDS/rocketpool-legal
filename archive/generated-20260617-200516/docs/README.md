# Docs Index

This folder contains the canonical working documentation for a Rocket Pool pDAO legal service-layer project.

These documents are written for non-lawyers. They should help the pDAO decide what to ask counsel, what problem it is trying to solve, and what language to use publicly. They should not be treated as legal or tax advice.

## Document map

- `00-project-overview.md` — project purpose, scope, and assumptions.
- `01-executive-summary.md` — short practical summary.
- `02-current-pdao-functions.md` — current committees and roles that may need support.
- `03-risk-model.md` — what risks the structure is meant to reduce.
- `04-entity-options-comparison.md` — DUNA vs LLCs, DAO LLCs, Swiss Association, Cayman, and doing nothing.
- `05-duna-service-layer.md` — how a DUNA-like service layer could work.
- `06-duna-membership-and-pseudonymity.md` — role-based protection and pseudonymous participation.
- `07-duna-scope-guardrails-and-tokenomics-firewall.md` — limits to preserve the service-layer framing.
- `08-third-party-funds-and-incentives.md` — special concerns around outside incentive funding.
- `09-tax-and-accounting-questions.md` — tax issues that need professional review.
- `10-language-guide.md` — words to use and avoid.
- `11-open-questions-for-counsel.md` — consolidated counsel checklist.

## Working principle

The pDAO remains the governance layer. The legal structure, if adopted, should be a narrow service layer for committee and administrative work.

## Canonical status

This folder should contain the complete current working analysis. There is no separate memo that must be read to understand the project. Files in `../drafts/` are optional implementation or communication drafts derived from this folder and should not be treated as a separate source of truth. If a draft introduces a new point, that point should be moved into the appropriate `docs/` file.
